# Git Workflow NexaTech

## Branching Strategy
- main: Production branch
- develop: Development branch

## Workflow
1. Developer membuat branch dari develop
2. Melakukan perubahan
3. Membuat Pull Request ke develop
4. Setelah stabil, merge ke main melalui Pull Request

## Rules
- Tidak diperbolehkan push langsung ke main
- Semua perubahan melalui Pull Request

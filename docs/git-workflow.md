# Git Workflow NexaTech

## Branch Strategy
- main: production branch
- develop: development branch

## Workflow
1. Developer melakukan perubahan di branch develop
2. Commit dan push ke develop
3. DevOps Engineer membuat Pull Request ke main
4. Pull Request direview
5. Merge ke main
6. Deployment otomatis ke GitHub Pages

## Rules
- Direct push ke main tidak diperbolehkan
- Semua perubahan harus melalui Pull Request
